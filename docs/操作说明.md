# FDE Knowledge Hub - 操作说明

## 环境准备

### 1. 打开 CMD 并进入项目目录

按 `Win + R`，输入 `cmd`，回车，然后：

```cmd
cd /d "D:\CodeX\FDE学习资料"
```

### 2. 创建并激活虚拟环境

```cmd
python -m venv venv
venv\Scripts\activate
```

激活后命令行前面会出现 `(venv)` 前缀。每次重新打开 CMD 都需要重新执行这步。

### 3. 安装依赖

```cmd
pip install -e . -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 功能测试流程

| 优先级 | 命令 | 依赖 |
|--------|------|------|
| 1 | `fde-cli status` | 无 |
| 2 | `fde-cli interview --count 3` | MiniMax API |
| 3 | `fde-cli path --hours 10` | MiniMax API |
| 4 | `fde-cli review` | 无 |
| 5 | `fde-cli ingest` | SiliconFlow + Qdrant |

### 第一步：系统状态检查

```cmd
fde-cli status
```

查看 Obsidian Vault、配置文件、知识条目统计是否正常。

### 第二步：模拟面试（核心 AI 功能）

```cmd
fde-cli interview --role FDE --difficulty senior --count 3
```

用 MiniMax 生成面试题，输入 `y` 进入练习模式可查看参考答案。报告自动保存到 `docs/interview_reports/`。

### 第三步：生成学习路径

```cmd
fde-cli path --topic fde --hours 10
```

调用 MiniMax 分析技能缺口，输出分阶段学习计划。

### 第四步：间隔复习

```cmd
fde-cli review --review-type adaptive
```

根据遗忘指数和重要性权重，列出今日应复习的条目。

### 第五步：向量知识摄取（需先启动 Qdrant）

```cmd
:: 启动本地 Qdrant
start vendor\qdrant\qdrant.exe

:: 验证连通性（需要 curl，或浏览器打开 http://localhost:6333）
curl http://localhost:6333/

:: 摄取 AI 工程化笔记
fde-cli ingest --category ai-engineering
```

### 第六步：递归补洞学习器

```cmd
python automation/gap_filling.py
```

输入学习主题后，从三个内置问题中选择切入角度：
- `[1]` 该领域专家共识的五个思维模式
- `[2]` 专家争论最激烈的三个方面
- `[3]` 10道区分真理解与死记硬背的测试题

程序自动检索本地知识库（优先 Qdrant 向量库，降级到 Markdown 文件），组装上下文后与 MiniMax AI 进行真实多轮对话。

输入「我已经完全理解」可将该主题存入 `docs/mastered_knowledge.json`，下次启动时该主题不再出现在上下文中。

## 常见问题

- **`python` 不是内部命令**：去 [python.org](https://python.org) 安装 Python，安装时勾选 **Add Python to PATH**
- **`fde-cli` 不是内部命令**：检查 `(venv)` 前缀是否存在，没有则先运行 `venv\Scripts\activate`
- **依赖安装失败**：尝试升级 pip：`python -m pip install --upgrade pip`，再重新安装
- **Qdrant 连接失败**：确认 `vendor\qdrant\qdrant.exe` 已启动，默认端口 6333
