## 📚 知识框架

### 1. Transformer 架构（"Attention is All You Need"，2017）

#### 整体结构
```
编码器（Encoder）：双向注意力，理解上下文
  输入嵌入 + 位置编码
  → N × (多头自注意力 + 前馈网络 + 残差+LayerNorm)

解码器（Decoder）：单向（因果）注意力，生成文本
  → 掩码自注意力（因果）
  → 交叉注意力（关注编码器输出）
  → 前馈网络
  → 输出 Softmax → 词表概率
```

#### 自注意力详解
```python
# Q: 查询（当前 token 想要什么）
# K: 键（其他 token 有什么）
# V: 值（实际内容）

scores = Q @ K.T / sqrt(d_k)     # 缩放点积，防止梯度消失
scores = softmax(scores)          # 归一化为注意力权重
output = scores @ V               # 加权聚合
```

#### 多头注意力
- 多个注意力头并行，每头关注不同的表示子空间
- `MultiHead(Q,K,V) = Concat(head₁,...,headₕ)Wᴼ`
- 不同头可以学习：语法关系、语义相似度、指代关系

#### 位置编码
- 原始 Transformer：固定正弦位置编码
- BERT/GPT：可学习的绝对位置编码
- **RoPE（旋转位置编码）**：LLaMA/GPT-NEO-X 使用；相对位置信息更外推能力强
- **ALiBi**：线性注意力偏置，更好的长度外推

### 2. 语言模型预训练

#### 训练目标分类
| 目标 | 代表模型 | 适用任务 |
|------|---------|---------|
| **自回归（CLM）** | GPT 系列 | 文本生成 |
| **掩码语言模型（MLM）** | BERT | 理解/分类 |
| **Seq2Seq** | T5, BART | 翻译/摘要 |

#### 缩放法则（Scaling Laws，Kaplan et al. 2020）
```
Loss = C / (N^α + D^β)

模型参数 N、数据量 D、计算量 C 三者同等重要
给定计算预算，最优分配约为：
  - Chinchilla 法则：N(参数) ≈ D(token数) 时训练最高效
  - GPT-3 时代普遍"参数过多训练不足"
```

### 3. Transformer 主要变体

| 模型 | 架构 | 特点 |
|------|------|------|
| **BERT** | Encoder-only | 双向，理解任务，需要微调 |
| **GPT 系列** | Decoder-only | 单向，生成，In-Context Learning |
| **T5** | Encoder-Decoder | 统一框架，一切任务转为文本生成 |
| **LLaMA** | Decoder-only | 开源，RoPE，RMSNorm，高效 |
| **Mistral** | Decoder-only | 滑动窗口注意力，Grouped Query Attention |

### 4. 指令微调（Instruction Tuning）与对齐

#### SFT（监督微调）
- 收集（指令，期望输出）对
- 在预训练模型上继续微调
- 让模型学会"按照指令行事"
- 代表：InstructGPT、Alpaca、LLaMA-Chat

#### RLHF（人类反馈强化学习）
```
1. 收集人类偏好数据（比较两个回答的好坏）
2. 训练奖励模型（Reward Model）
3. 用 PPO 强化学习优化语言模型
   目标：max E[RM(x,y)] - β·KL(π || π_ref)
   （最大化奖励，同时防止偏离原始模型太远）
```

#### DPO（直接偏好优化，2023）
- 跳过独立 RM，直接从偏好数据优化策略
- 更简单稳定；被 LLaMA3/Mistral 广泛采用

### 5. In-Context Learning 与 Prompting

#### 提示方式演进
| 方式 | 示例 | 效果 |
|------|------|------|
| Zero-shot | 直接问 | 基础能力 |
| Few-shot | 给 3-5 个例子 | 显著提升 |
| **CoT（思维链）** | "让我们逐步思考..." | 推理任务大幅提升 |
| **Self-Consistency** | 多次采样取投票 | 提升 CoT 准确率 |
| **ReAct** | 思考 + 行动交替 | 工具调用 |

#### Prompt 工程原则
1. **清晰具体**：任务描述、格式要求、约束条件都要明确
2. **角色设定**："作为一个专业的..."
3. **分步引导**：复杂任务分解为步骤
4. **示例驱动**：Few-shot 比说明更直接
5. **输出格式**：指定 JSON/Markdown 等结构化输出

### 6. 高效微调（PEFT）

#### LoRA（Low-Rank Adaptation，2021）
```python
# 原始权重矩阵 W (d×d)
# LoRA：W' = W + BA（B:d×r, A:r×d，r << d）

# 只训练 A 和 B，参数量从 d² 降到 2dr
# r=16, d=4096 时：4096² → 2×4096×16 = 256倍压缩
```

#### QLoRA
- 将基础模型量化为 4-bit（NF4），LoRA 适配器保持 BF16
- 单卡可微调 65B 参数模型
- 使内存需求降低 75%

#### 其他 PEFT 方法
- **Prefix Tuning**：在 KV 缓存前添加可学习向量
- **Adapter**：在 Transformer 层间插入小型网络
- **Prompt Tuning**：只优化输入 Prompt 的嵌入

### 7. LLM 推理优化

#### KV Cache
- 缓存已计算的 Key/Value，自回归生成时避免重复计算
- 长序列推理的显存瓶颈

#### 量化
| 精度 | 显存占用 | 质量损失 |
|------|---------|---------|
| FP32 | 4 byte/param | 无 |
| FP16/BF16 | 2 byte/param | 微小 |
| INT8 | 1 byte/param | 较小（需要校准）|
| INT4/GPTQ | 0.5 byte/param | 可接受 |

#### 投机采样（Speculative Decoding）
- 用小模型快速生成草稿，大模型批量验证
- 延迟降低 2-3x，不损失质量

## 🔑 关键概念

### Hallucination（幻觉）
**定义**: 模型生成看似合理但实际错误或虚构的内容

**重要性**: LLM 最核心的可靠性挑战；FDE 落地时客户最关心的问题

**应用场景**:
1. RAG（检索增强）：用检索到的事实锚定生成内容
2. 自一致性检查：多次采样比较结果
3. 引用来源：要求模型标注依据

---

### Context Window 与 Long Context
**定义**: 模型一次能处理的最大 Token 数

**重要性**: 直接影响可以处理的文档长度和对话历史

**应用场景**:
1. 128K+ 上下文：直接放入长文档（GPT-4o、Claude）
2. 超长文档：RAG 检索 + 短上下文推理
3. 对话历史：压缩摘要旧轮次

## 💡 实战应用场景

### 场景 1: 选择基础模型用于企业 RAG
**背景**: 客户需要搭建私有化 RAG 系统

**解决方案**: 按需选型开源/闭源模型

**实施步骤**:
1. 明确约束：数据隐私要求 + 推理成本 + 延迟要求
2. 私有化：LLaMA3/Qwen/Mistral + 量化部署（vLLM + GPTQ）
3. 云端：GPT-4o / Claude 3.5 Sonnet API
4. 评估：用真实业务问题做 A/B 测试

**预期效果**: 匹配业务需求的模型选型，有依据地说明权衡

---

### 场景 2: 提升 RAG 回答准确率
**背景**: RAG 系统频繁出现答非所问或幻觉

**解决方案**: 提示工程 + 答案接地性增强

**实施步骤**:
1. 改进检索：BM25 + 向量检索混合，提升召回质量
2. 加强 Prompt：明确要求"只基于提供的上下文回答，如无相关信息则说不知道"
3. 添加来源引用：要求模型标注答案来自哪段文本
4. 置信度过滤：低置信度回答触发人工审核

**预期效果**: 幻觉率降低 50-70%
