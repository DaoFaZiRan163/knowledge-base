# LLM 记忆分层架构：解决多轮对话体验断裂的工程实践

## 🎯 核心问题

LLM 的 context window 限制导致 AI"忘记"对话历史，造成多轮对话体验断裂。

## 🏗️ 三层记忆架构

| 层级 | 名称 | 存储方式 | 触发条件 |
|------|------|---------|---------|
| **L1** | 短期会话记忆 | 直接放入 context | 最近 12 轮对话 |
| **L2** | 长期向量记忆 | SQL Server 2025 向量存储 | 语义检索召回 |
| **L3** | 用户画像 | 结构化数据 | 注入 system prompt |

### L1 - 短期会话记忆 (Short-term Session Memory)

**机制：**
- 存储最近 12 轮对话直接放入 context
- 当 window 快满时，通过轻量级 summarization LLM 触发压缩

**保留关键信息：**
- 订单编号
- 问题类型
- 未解决项
- 顺序信息

### L2 - 长期向量记忆 (Long-term Vector Memory)

**机制：**
- 将对话摘要存储为语义向量
- 使用 bge-m3 进行嵌入
- 通过语义检索召回历史上下文

**能力：** 让 AI 能识别跨会话的重复问题

### L3 - 用户画像 (User Profile)

**存储内容：**
- 会员等级
- 语言偏好
- 常见问题类别

**更新策略：** 每次对话后增量更新

**注入方式：** 作为 system prompt 提供个性化上下文

## 📊 产品设计决策

| 决策 | 标准 |
|------|------|
| 设计目标 | "用户应该感知不到窗口切换" |
| UAT 测试标准 | 100 个测试用例中 ≤5% 感知到碎片化 |
| 多租户隔离 | JWT + 行级安全 |

## 📈 落地效果

| 指标 | 数值 |
|------|------|
| 会员自助解决率 | 65%（原来约 0%） |
| 工单量降低 | 30% |
| 用户满意度 | 87% |

## 💡 核心洞见

> "LLM 记忆问题本质上是产品体验问题，不只是技术问题。"

**架构优先级：** 用户感知 > 技术优化

通过智能记忆管理实现无缝对话，而非单纯追求最大化 context window。
