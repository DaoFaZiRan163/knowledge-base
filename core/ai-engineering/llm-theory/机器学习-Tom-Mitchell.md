# 《机器学习》- Tom Mitchell

## 📖 基本信息
- **作者**: Tom M. Mitchell
- **出版年份**: 1997
- **核心主题**: 机器学习的基础理论框架——从概念定义到核心算法的系统化入门
- **目标读者**: ML 入门者，建立理论基础的学习者
- **阅读难度**: intermediate

## 🎯 核心价值
> "如果一个程序在任务 T 上，随着经验 E 的积累，性能度量 P 得到提升，则称该程序从经验 E 中学习。" — 机器学习的经典定义，指向了设计 ML 系统的三个核心问题。

## 📚 知识框架

### 主要章节/模块

### 1. 机器学习核心概念

#### 学习类型
| 类型 | 定义 | 典型算法 |
|------|------|---------|
| **监督学习** | 有标注训练数据，学习输入→输出映射 | 线性回归、决策树、SVM、神经网络 |
| **无监督学习** | 无标注，发现数据内在结构 | K-Means、PCA、自编码器 |
| **强化学习** | 通过与环境交互和奖励信号学习策略 | Q-Learning、Policy Gradient |
| **半监督学习** | 少量标注 + 大量未标注数据 | 标签传播、自训练 |

#### 关键术语
- **特征（Feature）**：输入变量的表示
- **标签（Label）**：监督学习的目标变量
- **假设空间（Hypothesis Space）**：算法可能输出的所有函数集合
- **泛化（Generalization）**：在未见数据上的性能

### 2. 决策树（Decision Tree）
- **信息增益**：按哪个特征分裂能最大降低不确定性（ID3 算法）
- **基尼系数**：CART 算法使用；不需要计算 log
- **过拟合**：树太深，记住了训练集噪声
- **剪枝（Pruning）**：预剪枝（提前停止）/ 后剪枝（先长后剪）
- **随机森林**：多棵决策树集成，降低方差

### 3. 偏差-方差权衡（Bias-Variance Tradeoff）
```
总误差 = 偏差² + 方差 + 不可约误差

偏差（Bias）：模型假设与真实规律的差距 → 欠拟合
方差（Variance）：对训练数据的敏感度 → 过拟合
```
- 简单模型：高偏差、低方差（欠拟合）
- 复杂模型：低偏差、高方差（过拟合）
- 找到"甜点"：交叉验证确定最优复杂度

### 4. 评估方法
- **训练集 / 验证集 / 测试集**：7:1:2 或 8:1:1 拆分
- **K-折交叉验证**：数据量小时的标准评估方法
- **混淆矩阵**：TP / FP / TN / FN
- **指标选择**：
  - 准确率（Accuracy）：类别均衡时有效
  - 精确率（Precision）：假正例代价高时用（垃圾邮件）
  - 召回率（Recall）：假负例代价高时用（疾病诊断）
  - F1 = 精确率和召回率的调和平均
  - AUC-ROC：不同阈值下综合性能

### 5. 线性回归与逻辑回归
- **线性回归**：连续值预测；最小二乘法；正则化（L1/L2）
- **逻辑回归**：二分类；Sigmoid 函数将实数映射到概率
- **正则化**：L1（Lasso，稀疏解）/ L2（Ridge，权重收缩）/ Elastic Net
- **梯度下降**：批量 / 随机（SGD）/ 小批量（Mini-batch）

### 6. 支持向量机（SVM）
- 找到最大化间隔的分类超平面
- **支持向量**：离超平面最近的训练样本，决定边界位置
- **核技巧（Kernel Trick）**：通过核函数将数据映射到高维，处理非线性
- 常用核：线性核 / RBF 核（高斯）/ 多项式核

### 7. 神经网络基础（为深度学习打基础）
- **感知机**：单层线性分类器
- **多层感知机（MLP）**：隐藏层引入非线性（激活函数）
- **反向传播**：链式法则计算每个参数的梯度
- **激活函数**：Sigmoid / Tanh / ReLU（现代首选，解决梯度消失）

### 8. 贝叶斯学习
- **朴素贝叶斯（Naive Bayes）**：特征条件独立假设；文本分类的强基线
- **贝叶斯信念网络**：有向无环图表示变量条件依赖关系
- **最大似然估计（MLE）vs 最大后验估计（MAP）**：MAP = MLE + 先验

## 🔑 关键概念

### 没有免费的午餐定理（No Free Lunch）
**定义**: 没有一个算法在所有问题上都最优；每个算法都有其适用假设

**重要性**: 不要迷信"最好的算法"，要根据数据特征和问题性质选择

**应用场景**:
1. 结构化数据小样本 → 先试逻辑回归/SVM/梯度提升树
2. 图像/文本/序列 → 深度学习
3. 无标注数据探索 → 无监督方法

---

### 特征工程
**定义**: 将原始数据转化为算法可以有效利用的特征的过程

**重要性**: "数据和特征决定了机器学习的上限，算法只是在逼近这个上限"

**应用场景**:
1. 归一化/标准化（对距离敏感的算法必须做）
2. 类别特征 One-Hot 编码
3. 时间特征提取（小时、星期几、是否节假日）

## 💡 实战应用场景

### 场景 1: 客户要求构建预测模型
**背景**: 客户想预测用户是否会流失（Churn Prediction）

**解决方案**: 监督学习完整流程

**实施步骤**:
1. 确定标签：过去 30 天无活跃 = 流失
2. 特征工程：行为频率、最近活跃时间、付费金额
3. 分层采样（正负样本可能不均衡）
4. 训练梯度提升树（如 XGBoost），用 AUC 评估
5. 找到最优分类阈值（根据业务代价）

**预期效果**: 提前识别高流失风险用户，触发挽留策略

---

### 场景 2: 模型在生产中效果下降
**背景**: 模型上线后 3 个月 AUC 从 0.85 降到 0.72

**解决方案**: 检查数据分布偏移（Data Drift）

**实施步骤**:
1. 对比训练集和当前线上数据的特征分布
2. 计算 PSI（Population Stability Index）
3. 若发生偏移，用新数据重新训练或增量更新

**预期效果**: 及时发现并修复模型退化
