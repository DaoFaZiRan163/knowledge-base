# Hermes Agent：从"会聊天的工具"到"会长大的工作系统"

## 🎯 核心定义

**Hermes Agent 架构** 是一种让 AI Agent 从单纯对话能力进化为持续自我改进工作系统的方法论。核心思想是：Agent 真正有价值的地方不是更会聊天，而是能把**规则、记忆、工具和复盘**沉淀成持续进化的工作系统。

> "Agent 真正有价值的地方，不是更会聊天，而是能把规则、记忆、工具和复盘沉淀成持续进化的工作系统。"

## 🏗️ 核心架构

### 驾驭工程 (Harness Engineering)

传统思路是优化模型本身，Hermes 的思路是**优化运行环境**：

```
模型能力 ← 运行环境质量
         ├── 系统规则
         ├── 工具权限
         ├── 记忆召回
         ├── 反馈机制
         └── 协作流程
```

**核心理念：** 与其追求更聪明的模型，不如打造更好的操作系统环境。

### 三层记忆系统

| 层级 | 功能 | 特点 |
|------|------|------|
| **Session Memory** | 处理即时上下文 | 当前对话有效 |
| **Persistent Memory** | 捕获用户/业务偏好 | 跨会话持久化 |
| **Skill Memory** | 保存可复用流程 | 沉淀为固定技能 |

### Hermes 学习循环

任务完成后，系统评估：
1. 什么值得记忆？
2. 什么可以提炼为 Skill？
3. 现有 Skill 是否需要更新？

## 🔧 实践框架

### 1. 项目规则 (Project Rules)

将以下内容文档化：
- 技术栈选择
- 目录结构规范
- 命名约定
- 代码风格
- 禁止事项

### 2. 可复用技能 (Reusable Skills)

将重复任务封装为固定流程：
- PDF 分析
- 博客生成
- PR 审查

### 3. 任务后复盘 (Post-Task Review)

完成任务后识别：
- 新增规则
- 用户偏好
- 值得保留的检查项

## ⚖️ 风险分级授权

Hermes 倡导**渐进式授权**，根据风险等级决定自动化程度：

| 风险等级 | 处理方式 | 说明 |
|----------|----------|------|
| **低风险** | 自动处理 | 不会造成不可逆影响 |
| **中风险** | 人工确认 | AI 建议 + 确认后执行 |
| **高风险** | 正式审查 | 涉及商业策略等重大决策 |

## 📊 与传统 Agent 的对比

| 维度 | 传统 Agent | Hermes Agent |
|------|-----------|--------------|
| **能力** | 对话能力 | 系统能力 |
| **记忆** | 仅 Session | 三层记忆体系 |
| **进化** | 静态 | 持续改进 |
| **规则** | 隐式 | 显式文档化 |
| **风险控制** | 无差别授权 | 分级授权 |

## 💡 FDE 应用场景

### 场景 1: 客户现场部署
- 将客户技术栈规范化为 Project Rules
- 沉淀常见操作流程为 Skills
- 通过 Persistent Memory 记住客户偏好

### 场景 2: 内部效率提升
- 自动化代码审查流程
- 文档生成标准化
- 知识积累系统化
