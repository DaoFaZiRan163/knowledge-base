# 混合检索与召回率优化

## 核心定义

**混合检索** (Hybrid Search)：结合多种检索方法（如向量检索 + 关键词检索）来提升召回质量。

**召回率** (Recall)：检索系统找到的相关文档占总相关文档的比例。

**核心理念**：单一检索方法难以覆盖所有查询类型，混合检索通过融合多种信号提升整体召回率。

## 🎯 一句话总结

> **混合检索 = 向量相似度 + 关键词匹配 + 重排序融合**

```
查询: "如何优化Transformer模型的推理速度"

向量检索: 语义相似 → ["模型加速方法", "推理优化技术", "Transformer架构"]
BM25检索: 关键词匹配 → ["Transformer速度", "推理延迟优化", "模型优化"]
             ↓
        RRF融合 (Reciprocal Rank Fusion)
             ↓
        最终结果排序
```

## 🔧 技术实现

### RRF 融合公式

```python
def reciprocal_rank_fusion(results_list, k=60):
    """
    RRF 融合多个检索结果
    k: 融合参数，通常 60
    """
    scores = defaultdict(float)

    for results in results_list:
        for rank, doc in enumerate(results):
            # RRF 评分 = 1 / (k + rank)
            scores[doc.id] += 1 / (k + rank)

    # 按评分排序
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

### 向量 + BM25 混合检索

```python
from rank_bm25 import BM25Okapi
import numpy as np

class HybridRetriever:
    def __init__(self, vector_db, corpus):
        self.vector_db = vector_db
        self.bm25 = BM25Okapi(corpus)

    def retrieve(self, query, k=10):
        # 1. 向量检索
        vector_results = self.vector_db.search(query, k=k*2)

        # 2. BM25 检索
        tokenized_query = query.split()
        bm25_scores = self.bm25.get_scores(tokenized_query)
        top_bm25_indices = np.argsort(bm25_scores)[-k*2:][::-1]
        bm25_results = [corpus[i] for i in top_bm25_indices]

        # 3. RRF 融合
        fused = self._rrf_fusion(vector_results, bm25_results, k=k)

        return fused

    def _rrf_fusion(self, vec_results, bm25_results, k=60):
        scores = {}
        for rank, doc in enumerate(vec_results):
            scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)
        for rank, doc in enumerate(bm25_results):
            scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)[:k]
```

### 重排序 (Reranker)

```python
from sentence_transformers import CrossEncoder

class Reranker:
    def __init__(self, model_name="cross-encoder/ms-marco-MiniLM-L-12-v2"):
        self.model = CrossEncoder(model_name)

    def rerank(self, query, documents, top_k=5):
        # Cross-Encoder 精细化排序
        pairs = [(query, doc) for doc in documents]
        scores = self.model.predict(pairs)

        # 按评分排序
        ranked = sorted(zip(documents, scores), key=lambda x: x[1], reverse=True)
        return ranked[:top_k]
```

## 💼 FDE 应用场景

### 场景 1: 企业文档问答

**痛点**: 纯向量检索对关键词查询（如"如何取消订阅"）效果差

**解决方案**: 向量 + BM25 混合检索 + 重排序

**效果**: 关键词类问题召回率从 65% 提升至 92%

### 场景 2: 法律条文检索

**痛点**: 语义相似但表述不同的条文难以召回

**解决方案**: 混合检索 + 法律领域 Embedding + 重排序

**效果**: 语义相关条文召回率提升 30%

### 场景 3: 代码搜索

**痛点**: 精确函数名搜索被向量检索忽略

**解决方案**: 精确匹配（BM25） + 语义相似（向量） + 代码专用 Embedding

**效果**: 代码搜索准确率提升 45%

## 📊 性能指标

| 指标 | 说明 | 优化目标 |
|------|------|----------|
| Recall@K | 前K个结果中相关文档的比例 | ↑ 越高越好 |
| MRR | 第一个相关文档的排名倒数均值 | ↑ 越高越好 |
| NDCG | 考虑位置的相关性评分 | ↑ 越高越好 |
| 检索延迟 | 单次检索的响应时间 | ↓ 越低越好 |

## ⚠️ 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 融合权重分配 | 不同检索方法权重难以调优 | RRF 自适应融合 |
| 重复结果 | 多检索方法返回相同文档 | 去重 + 分数叠加 |
| 延迟增加 | 多检索方法 + 重排序 | 并行检索 + 截断 |
