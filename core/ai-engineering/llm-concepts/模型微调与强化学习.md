# 模型微调与强化学习 — 核心概念

## 核心定义

**模型微调** (Fine-tuning)：在预训练模型基础上，用特定任务数据进一步训练，使模型适配特定领域或任务。

**强化学习** (Reinforcement Learning)：通过奖励信号引导模型学习最优策略的方法。

**核心理念**：微调让模型"学会"新任务，强化学习让模型"学会"符合人类偏好。

## 🎯 一句话总结

> **微调 = 让模型适配任务，强化学习 = 让模型符合人类偏好**

```
预训练模型 (通用能力)
    ↓
指令微调 SFT (任务能力)
    ↓
对齐微调 RLHF/DPO (人类偏好)
    ↓
垂直领域微调 (企业定制)
```

## 🏗️ 微调方法对比

### SFT (监督微调)

```
训练数据: (输入, 期望输出) 对
损失函数: 交叉熵损失
目标: 最小化模型预测与期望输出的差异

优点: 简单直接，效果稳定
缺点: 需要大量标注数据，可能过拟合
```

### RLHF (人类反馈强化学习)

```
训练流程:
1. 收集人类偏好数据 (比较哪个回答更好)
2. 训练奖励模型 (Reward Model)
3. 使用奖励模型进行 PPO 强化学习

优点: 更好地对齐人类偏好
缺点: 训练复杂，reward hacking 风险
```

### DPO (直接偏好优化)

```
核心思想: 不需要 reward model，直接用偏好数据优化

损失函数: 基于偏好概率比
优点: 训练更简单，避免 reward model 误差
缺点: 对数据质量要求高
```

## 🔧 技术实现

### SFT 基本实现

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b")

# 准备数据
train_dataset = load_instruction_dataset("your-data.jsonl")

# 训练配置
training_args = TrainingArguments(
    output_dir="./output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    learning_rate=2e-5,
    warmup_steps=100,
    logging_steps=10,
)

# 开始训练
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
)
trainer.train()
```

### DPO 实现

```python
from trl import DPOTrainer

dpo_trainer = DPOTrainer(
    model=model,
    ref_model=ref_model,
    train_dataset=preference_dataset,
    args=training_args,
    beta=0.1,  # KL 散度系数
)
dpo_trainer.train()
```

### RLHF 实现

```python
from trl import PPOTrainer

# 1. 奖励模型训练
reward_model = train_reward_model(preference_data)

# 2. PPO 强化学习
ppo_trainer = PPOTrainer(
    model=model,
    ref_model=ref_model,
    reward_model=reward_model,
    args=ppo_config,
)

for batch in dataloader:
    query_tensors = batch["query"]
    response_tensors = ppo_trainer.generate(**query_tensors)
    reward = reward_model.predict(response_tensors)
    ppo_trainer.step(query_tensors, response_tensors, reward)
```

## 💼 FDE 应用场景

### 场景 1: 医疗领域适配

**需求**: 医疗问答模型需要专业、准确、可解释

**方案**: SFT + 医疗数据微调 + RLHF 对齐医疗专家偏好

**挑战**: 医疗数据隐私、高质量标注成本

### 场景 2: 客服语气调优

**需求**: 电商客服需要友好、耐心、促进转化

**方案**: DPO 微调，学习客服对话偏好

**效果**: 客户满意度提升，用户投诉减少

### 场景 3: 代码生成对齐

**需求**: 代码助手需要生成可读、可维护的代码

**方案**: SFT + RLHF，学习程序员偏好

**效果**: 代码审查通过率提升

## ⚠️ 常见陷阱

| 陷阱 | 症状 | 解决方案 |
|------|------|----------|
| 灾难性遗忘 | 微调后通用能力下降 | 保留原模型能力 (LORA) |
| Reward Hacking | 模型作弊讨好 reward model | 多样化奖励、增加 KL 约束 |
| 过拟合 | 微调数据量少时过拟合 | 数据增强、正则化 |
