# 数据清洗与评测集构建

## 核心定义

**数据清洗**：对原始数据进行处理，去除噪声、错误和不一致，提升数据质量。

**评测集 (Benchmark)**：用于评估模型性能的标准化测试数据集。

**核心理念**：模型质量取决于训练数据质量和评测集可靠性，"垃圾进，垃圾出"。

## 🎯 一句话总结

> **好数据 + 好评测 = 可靠模型评估**

```
数据清洗 pipeline:
原始数据 → 去重 → 质量过滤 → 格式标准化 → 高质量数据
                ↓
            评测集构建:
            高质量数据 → 任务划分 → 标注 → 评测集
```

## 🏗️ 数据清洗流程

### 典型步骤

```
1. 去重 (Deduplication)
   - 近似重复检测 (MinHash/SimHash)
   - 精确匹配去重
   - 去除低信息量内容

2. 质量过滤 (Quality Filtering)
   - 语言识别 (去除非目标语言)
   - 长度过滤 (过短/过长)
   - 格式过滤 (HTML标签、代码混乱)
   - 毒性检测 (有害内容)

3. 内容过滤 (Content Filtering)
   - 隐私信息去除 (PII)
   - 版权内容处理
   - 重复模式去除

4. 格式标准化 (Normalization)
   - 编码统一 (UTF-8)
   - 换行/空格标准化
   - 结构化转换
```

## 🔧 技术实现

### 去重实现

```python
from datasketch import MinHash

def minhash_deduplicate(documents, threshold=0.8):
    """MinHash 去重"""
    minhashes = []
    for doc in documents:
        tokens = doc.split()
        m = MinHash(num_perm=128)
        for token in tokens:
            m.update(token.encode('utf8'))
        minhashes.append(m)

    # 两两比较
    duplicates = []
    for i in range(len(minhashes)):
        for j in range(i+1, len(minhashes)):
            if minhashes[i].is_ann_neighbor(minhashes[j], threshold):
                duplicates.append((i, j))

    return duplicates
```

### 质量过滤

```python
def quality_filter(text, min_length=50, max_length=50000):
    """简单质量过滤"""
    # 长度检查
    if len(text) < min_length or len(text) > max_length:
        return False

    # 语言检测 (假设使用 langdetect)
    if detect_language(text) != 'zh':
        return False

    # 毒性检测 (假设使用 toxicity model)
    if is_toxic(text):
        return False

    # 格式检查 (去除过多特殊字符)
    special_char_ratio = sum(1 for c in text if not c.isalnum()) / len(text)
    if special_char_ratio > 0.5:
        return False

    return True
```

### 评测集构建

```python
class BenchmarkBuilder:
    def __init__(self):
        self.tasks = []

    def add_task(self, task_name, task_data):
        """添加评测任务"""
        self.tasks.append({
            "name": task_name,
            "data": task_data,
            "metrics": self._get_metrics_for_task(task_name)
        })

    def _get_metrics_for_task(self, task_name):
        """根据任务类型确定评测指标"""
        if "classification" in task_name:
            return ["accuracy", "f1", "precision", "recall"]
        elif "generation" in task_name:
            return ["bleu", "rouge", "bertscore"]
        elif "reasoning" in task_name:
            return ["accuracy", "pass_rate"]
        else:
            return ["accuracy"]

    def evaluate(self, model):
        """执行评测"""
        results = {}
        for task in self.tasks:
            predictions = []
            for item in task["data"]:
                pred = model.predict(item["input"])
                predictions.append(pred)

            metrics = compute_metrics(
                predictions,
                [item["target"] for item in task["data"]],
                task["metrics"]
            )
            results[task["name"]] = metrics

        return results
```

## 💼 FDE 应用场景

### 场景 1: 垂直领域模型训练

**需求**: 构建医疗问答模型

**方案**:
1. 收集医疗文献、医患对话
2. 数据清洗：去除隐私信息、统一格式
3. 构建医疗问答评测集（需医学专家标注）

**效果**: 数据质量提升，模型问答准确率提升 25%

### 场景 2: 企业知识库问答

**需求**: 构建内部知识问答系统

**方案**:
1. 收集内部文档、FAQ、聊天记录
2. 数据清洗：去除敏感信息、格式化
3. 构建内部评测集（IT/HR/法务分类）

**效果**: 评测集覆盖核心场景，模型准确率达 85%+

### 场景 3: 模型对比实验

**需求**: 评估不同模型在特定任务上的表现

**方案**:
1. 构建标准化评测集（100-500 条）
2. 确定评测指标（准确率、延迟、成本）
3. 多次采样取平均，减少随机性

**效果**: 可靠的模型对比，避免"选择最好的而非最适合的"

## 📊 评测指标

| 任务类型 | 指标 | 说明 |
|----------|------|------|
| 分类 | Accuracy, F1, Precision, Recall | 标准分类指标 |
| 生成 | BLEU, ROUGE, BERTScore | 文本相似度指标 |
| 问答 | EM, F1 | 匹配度指标 |
| 推理 | Accuracy, Pass@K | 正确率指标 |
| 对话 | RAGAS, TLDR | 质量评分指标 |

## ⚠️ 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 评测集泄漏 | 训练数据包含评测集内容 | 严格数据隔离 |
| 标注不一致 | 标注员理解差异 | 多人标注、Kappa 检验 |
| 过拟合评测 | 针对评测集调优 | 留出测试集、多 Benchmark |
