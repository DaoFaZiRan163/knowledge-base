# 并发熔断与容灾降级

## 核心定义

**熔断器 (Circuit Breaker)**：当下游服务故障时，快速失败并阻止持续请求，避免雪崩效应。

**限流 (Rate Limiting)**：控制请求速率，防止系统过载。

**降级 (Degradation)**：在服务不可用时，提供有损但可用的替代功能。

**核心理念**：在 LLM 服务不可靠时，快速失败而非无限等待，保证系统整体可用性。

## 🎯 一句话总结

> **熔断 = 检测故障 + 快速失败；降级 = 功能减少但服务可用**

```
正常状态:  请求 → LLM服务 → 响应
熔断状态:  请求 → 熔断器(快速返回) → 降级响应
```

## 🏗️ 熔断器模式

```
┌─────────────────────────────────────────────────────────────┐
│                    熔断器状态机                             │
│                                                             │
│    ┌─────────┐   失败超阈值    ┌─────────┐   探测成功   ┌───┐│
│    │ CLOSED  │ ─────────────→ │  OPEN   │ ───────────→ │HALF│
│    │ 正常   │                │ 熔断中  │              │OPEN││
│    └─────────┘               └─────────┘              └───┘│
│         ↑                          │                       │
│         └──────────────────────────┘                       │
│              重置计数器              等待时间后              │
└─────────────────────────────────────────────────────────────┘

状态说明:
- CLOSED: 正常状态，请求通过，失败计数
- OPEN: 熔断状态，所有请求快速失败
- HALF_OPEN: 探测状态，允许少量请求探测恢复
```

## 🔧 技术实现

### 熔断器实现

```python
import time
from enum import Enum

class CircuitState(Enum):
    CLOSED = "closed"
    OPEN = "open"
    HALF_OPEN = "half_open"

class CircuitBreaker:
    def __init__(self, failure_threshold=5, timeout=60):
        self.failure_threshold = failure_threshold
        self.timeout = timeout  # 秒
        self.failures = 0
        self.last_failure_time = None
        self.state = CircuitState.CLOSED

    def call(self, func, *args, **kwargs):
        if self.state == CircuitState.OPEN:
            # 检查是否应该转换到 HALF_OPEN
            if time.time() - self.last_failure_time > self.timeout:
                self.state = CircuitState.HALF_OPEN
            else:
                raise CircuitOpenError("Circuit is OPEN")

        try:
            result = func(*args, **kwargs)
            self._on_success()
            return result
        except Exception as e:
            self._on_failure()
            raise e

    def _on_success(self):
        self.failures = 0
        self.state = CircuitState.CLOSED

    def _on_failure(self):
        self.failures += 1
        self.last_failure_time = time.time()
        if self.failures >= self.failure_threshold:
            self.state = CircuitState.OPEN
```

### 限流器实现

```python
import asyncio
from collections import defaultdict

class RateLimiter:
    def __init__(self, max_calls, period):
        self.max_calls = max_calls
        self.period = period
        self.calls = defaultdict(list)

    async def acquire(self, key="default"):
        now = time.time()
        # 清理过期的调用记录
        self.calls[key] = [t for t in self.calls[key] if t > now - self.period]

        if len(self.calls[key]) >= self.max_calls:
            wait_time = self.period - (now - self.calls[key][0])
            await asyncio.sleep(wait_time)

        self.calls[key].append(now)
```

## 💼 FDE 应用场景

### 场景 1: 高并发 API 服务

**需求**: 支持每秒 100+ 请求的 LLM API 服务

**方案**:
1. 限流：每个用户每秒最多 10 请求
2. 熔断：LLM 服务失败率 > 10% 时触发熔断
3. 降级：返回"服务繁忙，请稍后重试"

**效果**: 系统在 LLM 服务不稳定时仍可正常响应

### 场景 2: 多 LLM 提供商切换

**需求**: 主提供商标价低但不稳定，备用提供商更贵但稳定

**方案**:
1. 主提供商 + 熔断器
2. 熔断触发时自动切换到备用
3. 恢复后逐步切回主提供商

**效果**: 成本降低 30%，可用性提升至 99.9%

### 场景 3: 企业内网隔离场景

**需求**: 网络不稳定时仍能提供服务

**方案**:
1. 检测网络故障
2. 自动降级到本地规则引擎
3. 记录离线期间请求，恢复后批量处理

**效果**: 恶劣网络环境下仍保持核心功能可用

## 📊 关键指标

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 熔断触发次数 | 单位时间内熔断次数 | > 10次/分钟 |
| 降级率 | 触发降级的请求比例 | > 5% |
| 平均响应时间 | P99 响应时间 | > 30s |
| 错误率 | LLM 调用错误率 | > 1% |

## ⚠️ 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 熔断器误触发 | 阈值设置过低 | 调高阈值、增加延迟窗口 |
| 降级恢复慢 | 缓存失效后直接打爆服务 | 渐进式恢复 |
| 多服务雪崩 | 级联故障 | 隔离舱模式、独立熔断 |
