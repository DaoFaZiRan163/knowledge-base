# CoT — 思维链提示 (Chain-of-Thought Prompting)

## 核心定义

**CoT (Chain-of-Thought)** 是一种 Prompt 技术，通过引导 LLM 生成**中间推理步骤**而非直接给出答案，显著提升模型在复杂推理任务上的表现。

**核心理念**：像人类解决问题一样，让模型"先思考再回答"，展示推理过程而非仅输出结果。

## 🎯 一句话总结

> **CoT = 让模型展示推理过程，而非直接给出答案**

```
标准Prompt:  Q: 小明有5个苹果，给了小红2个，还剩几个？
             A: 3个

CoT Prompt:  Q: 小明有5个苹果，给了小红2个，还剩几个？
            A: 小明一开始有5个苹果，给出2个，所以还剩5-2=3个
               答案是 3 个
```

## 🏗️ 架构原理

### 核心机制

```
┌────────────────────────────────────────────────────────────┐
│                    标准 Prompt                             │
│  输入问题 → 模型 → 直接输出答案                            │
│  (适合简单问答，不适合复杂推理)                             │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│                    CoT Prompt                              │
│  输入问题 → [触发思考] → 推理步骤1 → 推理步骤2 → ... → 最终答案
│  (展示完整推理链，适合多步问题)                             │
└────────────────────────────────────────────────────────────┘
```

### CoT 触发方式

| 方式 | 描述 | 适用场景 |
|------|------|----------|
| Few-shot CoT | 提供包含推理过程的示例 | 需要明确格式 |
| Zero-shot CoT | 加上 "Let's think step by step" | 快速启用 |
| Auto-CoT | 自动从数据集中采样示例 | 大规模应用 |

### Few-shot CoT 示例

```python
# 标准 Prompt
"""
Q: 小明有10块钱，买了3块钱的面包，还剩多少？
A: 7
"""

# CoT Prompt (包含推理过程)
"""
Q: 小明有10块钱，买了3块钱的面包，还剩多少？
A: 小明原有10元，消费3元，10-3=7，还剩7元。答案是7。

Q: 小红有15个糖果，分给4个朋友，每个朋友2个，还剩多少？
A: 小红分出4×2=8个糖果，15-8=7，还剩7个。答案是7。

Q: 小华买了两本书，一本12元，一本8元，付了50元，应该找零多少？
A: (待推理...)
"""
```

## 🔧 技术实现

### Zero-shot CoT

```python
# 最简单的启用方式
response = llm.invoke("""
用户问题：某商品原价100元，打8折后再减20元，最后多少钱？

让我们一步一步思考：
""")
```

### Few-shot CoT 实现

```python
# 完整示例
cot_prompt = """
请按照以下格式回答问题：

Q: [问题]
思考过程：
1. [第一步推理]
2. [第二步推理]
3. [第三步推理]
最终答案：[答案]

示例：
Q: 小明有5个苹果，给了小红2个，还剩几个？
思考过程：
1. 小明原有5个苹果
2. 给小红2个，用减法
3. 5 - 2 = 3
最终答案：3个

Q: 图书馆有120本书，学生借走了35本，又还回来28本，现在有多少本？
思考过程：
1. 原有120本书
2. 借走35本：120 - 35 = 85
3. 还回来28本：85 + 28 = 113
最终答案：113本

Q: 一件商品原价200元，打9折后再优惠50元，最后多少钱？
"""
```

### 与 DSPy 结合

```python
import dspy

# 使用 DSPy 的 ChainOfThought Module
cot = dspy.ChainOfThought(
    Signature.from_components(
        question="str",
        reasoning="str",  # 显式输出推理过程
        answer="str"
    )
)

result = cot(question="鸡兔同笼，共20个头，50只脚，问鸡兔各几只？")
print(f"推理: {result.reasoning}")
print(f"答案: {result.answer}")
```

## 💼 FDE 应用场景

### 场景 1: 金融风险评估

**客户需求**: AI 辅助评估贷款申请人的风险等级

**FDE 分析**:
- **痛点**: 直接给答案无法解释推理过程，不满足合规要求
- **机会**: CoT 展示评分逻辑，满足监管可解释性要求
- **风险**: 推理步骤可能泄露敏感信息

**实施策略**:
1. 设计风险评估的 CoT 模板
2. 分步展示：收入核实 → 负债计算 → 信用评分 → 最终结论
3. 加入人工复核环节

**预期效果**: 可解释的 AI 风险评估，通过合规审计

### 场景 2: 代码 Bug 诊断

**客户需求**: AI 自动诊断代码问题并给出修复建议

**FDE 分析**:
- **痛点**: 直接给修复方案用户不理解，根本问题未解决
- **机会**: CoT 展示从"症状观察"到"根因分析"到"修复方案"的完整链路
- **风险**: 推理错误可能误导开发者

**实施策略**:
1. 设计代码诊断的 CoT 模板
2. 步骤：复现问题 → 分析调用栈 → 定位根因 → 提出修复 → 验证方法

**预期效果**: 提升开发者对 AI 建议的信任度，减少无效沟通

### 场景 3: 复杂客服对话

**客户需求**: 处理需要多步推理的客服问题（如退款金额计算）

**FDE 分析**:
- **痛点**: 简单问答无法处理复杂场景，客诉率高
- **机会**: CoT 分步计算，减少计算错误
- **风险**: 推理步骤过多增加响应延迟

**实施策略**:
1. 识别需要 CoT 的对话类型
2. 条件触发：涉及多步计算时自动启用
3. 简化输出：内部详细推理，外部简洁呈现

**预期效果**: 复杂退款场景处理正确率从 75% 提升至 95%

## ⚠️ 常见陷阱与解决方案

### 陷阱 1: CoT 推理错误累积
**症状**: 早期推理步骤错误，导致最终答案错误但模型"自信满满"

**原因分析**: 模型在生成推理步骤时会"坚持"早期错误表述

**解决方案**:
1. 在 Prompt 中加入"检查推理一致性"步骤
2. 使用 [[Self-Consistency]] 多次采样投票
3. 关键步骤加入验证环节

### 陷阱 2: 推理步骤过多导致输出冗长
**症状**: 用户只想要答案，但模型输出了大段推理过程

**原因分析**: CoT 格式过于僵化

**解决方案**:
1. 区分"内部推理"和"外部输出"
2. 提供可折叠的推理过程展示
3. 使用"先推理后总结"的双阶段输出

### 陷阱 3: 简单问题强制 CoT 反而降低效率
**症状**: 简单问答加上 CoT 后响应时间显著增加

**原因分析**: 不是所有问题都需要 CoT

**解决方案**:
1. 问题复杂度预判（可用小模型先判断）
2. 简单问题直接回答，复杂问题启用 CoT
3. 用户可手动关闭 CoT

## 📊 性能优化

### 性能指标
- **延迟增加**: CoT 相比直接回答增加 30-100% token 输出
- **准确率提升**: 数学推理提升 15-30%，代码推理提升 10-20%
- **成本**: 输出 token 增加，成本相应上升

### 优化策略
1. **自适应 CoT**: 简单问题跳过推理步骤
2. **模板简化**: 推理步骤使用简洁缩写而非完整句子
3. **流式输出**: 推理过程分步流式返回，用户可提前终止

## 🔍 FAQ

### Q1: CoT 适用于所有任务吗？

不。CoT 主要提升**多步骤推理**任务的效果：
- ✅ 数学问题、逻辑推理、代码调试
- ✅ 需要解释的决策场景
- ❌ 简单事实问答、情感分析、文本分类（效果提升有限）

### Q2: Few-shot 和 Zero-shot CoT 怎么选？

| 场景 | 推荐方式 |
|------|----------|
| 小规模、高准确率要求 | Few-shot（10-20条示例）|
| 大规模、低延迟要求 | Zero-shot（"Let's think step by step"）|
| 动态场景 | Auto-CoT（自动生成示例）|

### Q3: CoT 与 System Prompt 的关系？

CoT 通常作为 User Prompt 的一部分，也可以整合到 System Prompt 中。建议：
- 固定推理模式 → 放 System Prompt
- 一次性推理 → 放 User Prompt
