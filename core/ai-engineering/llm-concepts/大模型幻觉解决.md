# 大模型幻觉 — 问题成因与解决方案

## 核心定义

**幻觉 (Hallucination)**：大模型生成的内容看似流畅合理，但包含错误、过时或不存在的信息，模型"自信地胡说八道"。

**核心理念**：LLM的本质是"概率预测"而非"事实检索"，它生成的是最可能的文本，而非最正确的文本。

## 🎯 一句话总结

> **幻觉 = 模型在"编造"而非"回忆"，根本原因是无知识边界 + 概率生成 + 训练数据偏差**

## 📊 幻觉类型分类

| 类型 | 描述 | 示例 |
|------|------|------|
| 事实性幻觉 | 生成与事实不符的信息 | "秦始皇统一六国是1911年" |
| 语义幻觉 | 前后语义不一致 | 前文说A，后文否定A |
| 编排性幻觉 | 无中生有的细节 | 引用不存在的论文/数据 |
| 领域幻觉 | 专业领域知识错误 | 医疗/法律胡说八道 |

## 🔍 幻觉成因分析

### 根本原因

```
LLM 生成过程:
输入 → [神经网络推理] → 概率分布 → 采样输出

问题:
1. 知识边界模糊 —— 模型不知道"不知道"
2. 概率采样 —— 总是选最可能的，而非最正确的
3. 训练数据偏差 —— 错误信息也被学进去了
4. 上下文混淆 —— 多轮对话中上下文冲突
```

### 训练阶段来源

| 阶段 | 引入的幻觉风险 |
|------|----------------|
| 预训练 | 互联网数据中错误信息被学进去 |
| SFT | 标注数据质量差，错误示范 |
| RLHF | 人类反馈中的偏好偏差 |
| 对齐微调 | 过度追求流畅性牺牲事实性 |

## 🛠️ 解决幻觉的核心方法

### 方法 1: RAG（检索增强生成）

**原理**：让模型"看着证据说话"，而非凭空生成。

```
用户问题
    ↓
检索知识库 (向量相似度)
    ↓
获取相关证据片段
    ↓
将证据作为上下文输入LLM
    ↓
LLM基于证据生成 (必须引用/使用证据)
```

**RAG 的幻觉抑制效果**：

| 无RAG | 有RAG |
|-------|-------|
| 自由发挥，可能胡说 | 有据可依，减少编造 |
| 知识截断（靠记忆） | 实时检索最新知识 |
| 无法验证来源 | 证据可溯源 |

**实现要点**：

```python
# RAG 幻觉抑制示例
def rag_answer(question, knowledge_base):
    # 1. 检索相关知识
    relevant_docs = knowledge_base.search(question, top_k=5)

    # 2. 检查置信度
    if relevant_docs.confidence < 0.7:
        return "我无法确定，请咨询专业人士"

    # 3. 明确指示模型使用证据
    prompt = f"""
    基于以下证据回答，不要超出证据范围：
    证据：{relevant_docs.content}
    问题：{question}
    如果证据不足以回答，明确说明"我不知道"
    """

    # 4. 输出加事实标签
    response = llm.generate(prompt)
    return f"{response} [来源: {relevant_docs.source}]"
```

### 方法 2: 提示工程 (Prompt Engineering)

#### 2.1 CoT (思维链) — 减少胡乱猜测

```
无CoT: 用户问 → 模型直接答（可能瞎猜）
有CoT: 用户问 → 模型先推理步骤 → 再给答案（更严谨）
```

```python
# CoT 提示减少幻觉
prompt = """
问题：我有多少钱？
先思考：
1. 我的收入来源是什么？
2. 我的支出有哪些？
3. 收入 - 支出 = ?
只有确定知道才回答，不知道就说不知道。
"""
```

#### 2.2 约束输出 (Constrained Decoding)

```python
# 限制输出格式，减少自由发挥
prompt = """
你是一个数学计算器，只能输出数字和运算符。
不要输出任何解释。
输入：2+3
输出：5
"""
```

#### 2.3 Self-Consistency (自洽性提示)

```python
# 让模型多次回答，取一致的结果
answers = [llm.ask(question) for _ in range(3)]
consistent_answer = pick_most_frequent(answers)
```

### 方法 3: 模型训练层面的解决

#### 3.1 RLHF / DPO — 对齐训练

```python
# 用人类反馈减少幻觉
# RLHF: 奖励模型惩罚错误输出
# DPO: 直接优化偏好（更简单）
```

| 方法 | 原理 | 幻觉改善效果 |
|------|------|-------------|
| RLHF | 奖励模型学人类偏好，惩罚幻觉输出 | 中等 |
| DPO | 直接对比正确vs错误，偏好学习 | 中等 |
| RLAIF | 用AI反馈替代人类反馈 | 待验证 |

#### 3.2 模型蒸馏与量化

```
大模型(100B+) → 小模型(7B) → 保留核心知识，减少噪声学习
```

#### 3.3 换更强模型

| 模型 | 幻觉率 (TruthfulQA) | 说明 |
|------|-------------------|------|
| GPT-3.5 | ~50% | 较高幻觉 |
| GPT-4o | ~20% | 显著改善 |
| Claude 3 | ~15% | 优秀 |
| GPT-4o-mini | ~25% | 性价比好 |

### 方法 4: 知识边界管理

#### 4.1 知识图谱增强

```
用户问 → 查询知识图谱 → 有答案 → 直接返回
                   → 无答案 → 模型回答或拒答
```

#### 4.2 置信度校准

```python
def calibrated_answer(question):
    response, confidence = llm.generate_with_confidence(question)

    if confidence < 0.6:
        return "我不确定，但可以告诉你..."
    elif confidence < 0.8:
        return f"{response} (仅供参考)"
    else:
        return response
```

#### 4.3 拒答机制

```
系统提示词加入：
"如果你不确定答案，请明确说'我不知道'，
不要编造。诚实比正确更重要。"
```

### 方法 5: 评测与监控

#### 5.1 幻觉评测 Benchmark

| Benchmark | 用途 | 评价指标 |
|-----------|------|----------|
| TruthfulQA | 事实性幻觉 | 准确率(越高越好) |
| FActScore | 生成内容事实性 | FActScore (越高越好) |
| HaluEval | 综合幻觉评估 | 幻觉检测率 |

```python
# 幻觉检测评估示例
def evaluate_hallucination(model, test_set):
    results = []
    for item in test_set:
        response = model.generate(item.question)
        is_hallucination = detector.check(response, item.ground_truth)
        results.append({
            "question": item.question,
            "response": response,
            "hallucination": is_hallucination
        })

    # 计算幻觉率
    hallucination_rate = sum(r["hallucination"] for r in results) / len(results)
    return hallucination_rate
```

#### 5.2 线上监控

```
生产环境:
├── 录音存储
├── ASR转写
├── 事实性校验 (对比知识库)
├── 幻觉率统计
└── 告警阈值
```

## 💼 实际生产组合方案

### 最优组合：多层防御

```
第1层: RAG(知识库检索) → 有证据才答
第2层: CoT(思维链推理) → 先想后答
第3层: 置信度过滤 → 不确定拒答
第4层: 输出加溯源标签 → 可验证
第5层: 人工抽检 → 质量兜底
```

### 不同场景的策略

| 场景 | 风险 | 推荐方案 |
|------|------|----------|
| 医疗咨询 | 高(人命关天) | RAG+拒答+人工复核 |
| 金融分析 | 高(投资决策) | RAG+知识图谱+置信度 |
| 客服问答 | 中 | RAG+CoT+人工抽检 |
| 娱乐创作 | 低 | 无特殊要求 |

## ⚠️ 常见误区

| 误区 | 正确做法 |
|------|----------|
| "模型知道就说，不知道就编" | 错误：要让模型主动说"不知道" |
| "用更大模型就能解决幻觉" | 部分有效，但无法根除 |
| "幻觉评测不重要" | 必须测，否则上线后出问题 |
| "一次解决就完事" | 幻觉需要持续监控和迭代 |
