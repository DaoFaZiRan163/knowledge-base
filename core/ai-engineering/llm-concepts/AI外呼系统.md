# AI外呼系统

## 核心定义

**AI外呼**：用语音AI技术（ASR + TTS + NLU）替代或辅助人工坐席，自动完成外呼任务的系统。

**核心理念**：把ASR/TTS/NLU串成流水线，加上批量外呼、话术管理、意向分级，实现7×24小时无人值守的规模化触达。

## 🎯 一句话总结

> **AI外呼 = 批量拨号 + 语音对话 + 意向识别 + CRM对接，替代人工做"筛客户"工作**

## 📊 架构演进

| 时代 | 技术 | 特点 | 效果 |
|------|------|------|------|
| 传统人工 | 纯人工拨打 | 成本高，效率低 | 100通/天/人 |
| IVR自动化 | 预设语音菜单 | 简单播报，无法对话 | 减人力，体验差 |
| 规则语音机器人 | 关键词匹配 | 固定话术，死板 | 可筛选，但机械 |
| AI对话机器人 | ASR+NLU+TTS | 智能对话，自然 | 可对话，但理解浅 |
| 大模型外呼 | LLM驱动 | 真正理解，多轮 | 接近人工 |

## 🏗️ 系统架构

### 完整AI外呼流水线

```
CRM/数据系统
     ↓
[1.名单管理] → 数据清洗 → 号码分级 → 拨打策略
     ↓
[2.外呼引擎] → 自动拨号 → 线路调度 → 接通检测
     ↓
[3.语音对话模块]
     ├── ASR: 语音 → 文本
     ├── NLU: 文本 → 意图+槽位
     ├── 对话管理: 状态机/LLM
     └── TTS: 文本 → 语音
     ↓
[4.意向识别] → A/B/C/D分级 → 实时写入CRM
     ↓
[5.转人工/挂机] → 高意向转人工 → 其他自动挂断
```

### 核心模块详解

#### 对话管理 (Dialogue Management)

| 方式 | 原理 | 适用场景 | 缺点 |
|------|------|----------|------|
| 状态机 | 预设节点+跳转规则 | 简单流程 | 无法处理未预设情况 |
| LLM驱动 | 大模型理解+生成 | 复杂多轮 | 延迟高，成本高 |
| 混合 | 状态机+LLM兜底 | 生产环境 | 复杂度高 |

#### 话术引擎

```
话术结构:
├── 开场白 (5秒内抓住注意力)
├── 价值传递 (产品亮点)
├── 异议处理 (预设+AI生成)
├── 邀约/成交 (明确行动号召)
└── 结束语 (挂机前最后印象)

支持:
├── SSML标记 (停顿、重音、语速)
├── 变量插入 (客户姓名、产品名)
└── 情绪检测 (自动调整语速语调)
```

## 💼 生产部署要点

### 性能指标要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 端到端延迟 | <1秒 | 从用户说话到AI回应 |
| ASR准确率 | >92% | 嘈杂环境可降至85% |
| 接通后持续时间 | >45秒 | 太短说明被打断 |
| 对话完成率 | >70% | 完成完整话术流程 |

### 线路与并发

| 规模 | 并发数 | 线路要求 |
|------|--------|----------|
| 小规模 | 10-50路 | SIP中继 + 语音网关 |
| 中规模 | 50-200路 | 云线路 (容联/梦网/云之讯) |
| 大规模 | 200路+ | 运营商直连 + 负载均衡 |

### 对话调优流程

```
1. 埋点采集 → 记录每轮对话文本+ASR置信度
2. 语料分析 → 聚类高频badcase
3. 话术更新 → 补充意图/回复策略
4. A/B测试 → 新旧话术效果对比
5. 全量上线 → 灰度发布
```

## ⚠️ 核心挑战与解决方案

| 挑战 | 描述 | 解决方案 |
|------|------|----------|
| 打断处理 | 用户随时打断AI说话 | VAD + 打断后停止当前输出 |
| 噪声环境 | 客户环境嘈杂 | 语音增强(SE) + 降噪ASR |
| 方言口音 | 各地方言识别难 | 方言ASR模型 + 纠错 |
| 对话深度 | 多轮复杂对话易迷路 | 对话状态跟踪 + LLM |
| 接通率低 | 号码无效/被标记 | 号码清洗 + 时段策略 |
| 监管合规 | 禁止AI伪装人工 | 明确告知 + 人工兜底 |
