## 📚 知识框架

### 主要章节/模块

### 1. 三大财务报表

#### 损益表（Income Statement / P&L）
```
收入（Revenue）
- 销售成本（COGS / Cost of Goods Sold）
= 毛利润（Gross Profit）
  毛利率 = 毛利润 / 收入

- 运营费用（OPEX: 销售、管理、研发）
= 运营利润（Operating Income / EBIT）

- 利息和税
= 净利润（Net Income）
```

**关键洞察**：
- EBITDA（息税折旧摊销前利润）= 衡量运营现金生成能力，排除非现金影响
- 毛利率反映定价能力和成本控制
- 净利润≠现金（应计制会计的影响）

---

#### 资产负债表（Balance Sheet）
```
资产 = 负债 + 股东权益

资产：
  流动资产：现金、应收账款、存货
  非流动资产：固定资产、无形资产

负债：
  流动负债：应付账款、短期借款
  非流动负债：长期债务

股东权益：
  实收资本 + 留存收益
```

**关键指标**：
- 流动比率 = 流动资产 / 流动负债（>2 为健康）
- 资产负债率 = 总负债 / 总资产（越低越安全）
- 净资产收益率（ROE）= 净利润 / 股东权益（越高越好）

---

#### 现金流量表（Cash Flow Statement）
```
经营活动现金流（Operating CF）：日常业务产生/消耗的现金
投资活动现金流（Investing CF）：资本开支（买设备/系统）
融资活动现金流（Financing CF）：借款、还款、分红

净现金流 = 三者之和
```

**关键洞察**：
- "利润好但现金流差"是企业死亡的常见原因（应收账款太多）
- 自由现金流（FCF）= 经营 CF - 资本开支 → 最重要的价值指标

---

### 2. 关键财务比率

#### 盈利能力
| 指标 | 公式 | 含义 |
|------|------|------|
| 毛利率 | 毛利/收入 | 产品本身的盈利能力 |
| 净利率 | 净利润/收入 | 整体运营效率 |
| ROE | 净利润/股东权益 | 股东投资回报率 |
| ROIC | 税后净利润/投入资本 | 资本利用效率 |

#### 流动性
| 指标 | 公式 | 健康水平 |
|------|------|---------|
| 流动比率 | 流动资产/流动负债 | >2 |
| 速动比率 | (流动资产-存货)/流动负债 | >1 |
| 现金比率 | 现金/流动负债 | >0.5 |

### 3. AI 项目的财务分析工具

#### ROI 计算
```
ROI = (净收益 / 投资成本) × 100%

净收益 = 节省成本 + 新增收入 - AI 运营成本

示例：
  投资成本：200万（开发 + 实施）
  年节省人力成本：120万
  年增加收入：50万
  年AI运营成本：30万

  净年收益 = 120 + 50 - 30 = 140万
  回收期 = 200/140 = 1.43年
  3年ROI = (140×3 - 200) / 200 = 110%
```

#### NPV（净现值）
```
NPV = Σ (现金流₍ₜ₎ / (1+r)^t) - 初始投资

r = 折现率（公司资本成本，通常 8-15%）
NPV > 0：项目创造价值
NPV < 0：项目销毁价值
```

#### TCO（总拥有成本）
```
TCO = 初始开发 + 实施 + 培训 + 年运营 × 年数 + 维护升级

AI 系统的隐性成本：
  - GPU 推理成本（按 API 调用或自建服务器）
  - 数据标注和维护
  - 模型定期重训练
  - 合规和安全审计
```

### 4. 与 CFO 对话

#### CFO 关心的四个问题
1. **花了多少钱？**（TCO，含隐性成本）
2. **什么时候收回？**（Payback Period）
3. **长期回报如何？**（NPV/IRR）
4. **风险有多大？**（敏感性分析）

#### 财务汇报的黄金句式
- "这个项目 18 个月回本，3 年 ROI 110%，假设折现率 10% NPV 约 X 万"
- "主要风险是用户采纳率，如果只达到 60% 目标，回收期延长至 24 个月"
- "与同类行业项目相比，我们的实施成本低 30%"

### 5. SaaS 业务财务指标

| 指标 | 公式 | 健康基准 |
|------|------|---------|
| MRR | 月经常性收入 | 持续增长 |
| ARR | MRR × 12 | — |
| Churn Rate | 流失 MRR / 上月 MRR | < 2%/月 |
| NRR（净收入留存）| (续约+扩容-流失)/原始MRR | > 100% |
| CAC | 获客总费用/新客户数 | — |
| LTV | ARPU / Churn Rate | LTV/CAC > 3 |

## 🔑 关键概念

### 应计制 vs 收付实现制
**定义**: 
- 收付实现制：实际收到/付出现金才记录
- 应计制：发生交易时就记录（无论现金何时流动）

**重要性**: 理解这个差异才能理解为什么"有利润但没现金"

**应用场景**:
1. 客户签单但未付款：损益表有收入，现金流没有
2. 预付年费：收到现金，但收入要分 12 个月摊销
3. AI 项目分期交付但一次性收费：需要识别收入确认时点

---

### EBITDA 的用途与局限
**定义**: 净利润加回利息、税、折旧、摊销

**重要性**: 剔除财务结构和会计政策差异，衡量经营层面的现金生成能力；PE 估值常用 EV/EBITDA

**注意事项**: EBITDA 不等于现金流，忽视了资本开支；不能完全替代 FCF

## 💡 实战应用场景

### 场景 1: 向 CFO 汇报 AI 项目价值
**背景**: FDE 需要向客户 CFO 证明 200 万 AI 项目的价值

**解决方案**: 标准化财务分析报告

**实施步骤**:
1. 建立 TCO 模型（列出所有成本项）
2. 量化效益（时间节省×人力成本 + 错误减少×质量成本）
3. 计算 ROI、回收期、NPV（用客户标准折现率）
4. 做敏感性分析：采纳率 50%/70%/100% 的三种情景

**预期效果**: CFO 可以直接用这些数据做投资决策

---

### 场景 2: 评估客户是否有 AI 采购预算
**背景**: 不确定客户是否真的有钱购买 AI 解决方案

**解决方案**: 从公开财务信息分析

**实施步骤**:
1. 看年报或新闻了解营收规模和利润率
2. 估算 IT/数字化预算（通常是营收的 3-8%）
3. 了解近年重大 IT 投资情况（避免刚刚大投资后预算耗尽）
4. 和客户聊 ROI 期望（<2 年回本是常见门槛）

**预期效果**: 提前判断机会质量，避免无效商机
